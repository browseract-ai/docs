## ğŸš€Integration Steps

### Step 1: Add BrowserAct Node to n8n

1. Open your n8n workflow (or create a new one)
2. Click the `+` button to add a node
3. Search for **"BrowserAct"**
4. Click to add the BrowserAct node to your workflow

### Step 2: Connect Your API Key

1. In the BrowserAct node, find the **Credentials** section
2. Click **"Create New Credential"**
3. Paste your BrowserAct API key
4. Click **Save** and select it from the dropdown

ğŸ’¡ **Test the connection:** Click the "Test" button next to your credential. You should see "Connection successful"

### Step 3: Choose Your Workflow

1. In the BrowserAct node, set **Resource** to `Workflow`
2. Set **Operation** to `Run a workflow`
3. Choose search source:
   1. **My Workflows** â€“ Your own workflows (most common)
   2. **Template Marketplace** â€“ Pre-built templates (optional)
1. Select your workflow from the dropdown list

### Step 4: Map Your Input Parameters

After you pick a workflow, n8n automatically shows you all input parameters that workflow needs (like "City", "URL", "Keyword", etc).

Now map data from previous nodes:

1. For each parameter field, click the input area
2. Choose **Mapping** mode (click the icon with 3 dots)
3. Select the field from your previous node:
   1. From Google Sheets? Pick the column name
   2. From webhook? Pick the field name
   3. From database? Pick the data field

Example mapping:

BrowserAct param "City" â† Sheets column "city"

BrowserAct param "Keyword" â† Webhook field "search_term"

BrowserAct param "URL" â† CRM field "website"

### Step 5: Test the Integration

1. Click the **"Test"** button (or press `Cmd/Ctrl + Enter`)
2. n8n will execute your BrowserAct workflow with test data
3. You'll see your scraped data in the output panel on the right
4. If you see results, you're ready! If there's an error, check troubleshooting below

### Step 6: Save Your Workflow

1. Click **"Save"** to save your workflow
2. Your n8n workflow is now ready to use!

## ğŸ†˜Troubleshooting

#### âŒ "API Key Invalid" or "Connection Failed"

n8n can't authenticate with BrowserAct

**Fix:** (1) Double-check your API key is correct (copy from BrowserAct Settings) (2) Make sure key is not expired or regenerated (3) Try creating a new API key if needed (4) Click "Test Connection" button in n8n to verify

#### âŒ "Workflow Not Found"

n8n can't find your BrowserAct workflow

**Fix:** (1) Make sure workflow is Published in BrowserAct (2) Refresh the workflow dropdown (sometimes cache needs refresh) (3) Check you're looking in "My Workflows" not "Templates" (4) Try re-authorizing your connection (delete and recreate credential)

#### âŒ "Parameter Mismatch" or "Field Not Found"

Input parameters don't match what BrowserAct expects

**Fix:** (1) Check parameter names are exact match (case-sensitive!) (2) Verify data types (string vs number vs boolean) (3) Make sure previous node outputs the field you're mapping (4) Use "Test" button to see the actual data being sent

#### âŒ Workflow Runs but Returns Empty/Null Results

BrowserAct runs but doesn't find data

**Fix:** (1) Test the workflow directly in BrowserAct first (does it work there?) (2) Check that parameters are being passed correctly (3) Verify the website hasn't changed (layouts change, elements move) (4) Check if you're hitting rate limits (too many runs too fast)

#### âŒ Timeout Error (Workflow takes too long)

Workflow execution exceeds time limit

**Fix:** (1) Check if website is slow (test workflow manually in BrowserAct) (2) Add wait/delay nodes in BrowserAct if page loads slowly (3) Some websites block automated access (try proxy, add delays) (4) Contact support if legitimate workflow times out
